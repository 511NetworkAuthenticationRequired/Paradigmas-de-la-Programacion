# UIMANAGER

## a.
```smalltalk
UIManager default request: '¿Cuál es tu edad ? '
```

La expresión hace uso de **UIManager**, la clase responsable en Pharo de gestionar cuadros de diálogo para interacción con el usuario. Justamente, el mensaje `request:` abre una ventana emergente solicitando un valor de texto. El argumento (`'¿Cuál es tu edad ? '`) es el mensaje mostrado en la interfaz.

1. `UIManager default` devuelve el gestor de interfaces por defecto.
2. `request:` despliega una ventana para que el usuario ingrese un valor.
3. El método devuelve la entrada del usuario como un **String**.  
   Ejemplo: si se escribe `25`, retorna `'25'`.

> [!CAUTION]
> - Cuando se usa UIManager siempre retorna un `String`, sin importar que se ingrese, por eso si se quiere operar con números debe convertirse mediante un (`asInteger`) al final. Para caracteres se puede usar `first`, que devuelve el primer caracter del string ingresado.
> - Si el usuario cancela la operación, se obtiene `nil`, lo que debe contemplarse para evitar errores. Con un `ifNil: [^self]` muchas veces basta para que no aparezca "una ventana de errror".

---

## b.
```smalltalk
| nombre |
nombre := UIManager default request: '¿Cómo te llamas? '.
^UIManager inform: 'Hola ', (nombre asString).
```

- Se solicita al usuario que ingrese su nombre mediante `request:`.
- La respuesta se almacena en la variable `nombre`.
- Luego se muestra un cuadro de información (`inform:`) saludando al usuario con el valor ingresado.

Como se menciono antes, el método `asString` asegura que el valor retornado se trate como texto. Al mismo tiempo, si el usuario cancela la ventana, `nombre` puede ser `nil`, lo que conviene manejar para evitar errores.

> [!NOTE]
> Se puede cambiar `inform:` por un `alert:` también para mostrar por pantalla, que en realidad muestra un mensaje de advertencia o error. Yo uso este porque te aparece en el medio de la pantalla si o si.

---

## c.
```smalltalk
| resp |
resp := UIManager confirm: '¿Hoy es tu cumpleaños??'.
resp ifTrue: [ UIManager inform: 'FELIZ CUMPLE!!' ]
```

- Se abre un cuadro de confirmación con `confirm:`, que devuelve un valor booleano (`true` o `false`).
- Si el usuario selecciona **Aceptar**, la condición se cumple y se muestra un mensaje de felicitación con `inform:`.

> [!NOTE]
> `confirm:` es útil cuando se necesita una respuesta **sí/no**, porque justamente te muestre opciones **yes/no**. También se puede usar para una difurcación en otros casos que no sean directamente un simple **si/no**, como un **opción 1/opción 2** pero eso no aparecera entre las opciones.

---

Estos ejemplos muestran cómo `UIManager` en Pharo permite interacciones básicas con el usuario:
- `request:` para pedir datos de entrada.
- `inform:` para mostrar mensajes.
- `confirm:` para preguntas de tipo sí/no.

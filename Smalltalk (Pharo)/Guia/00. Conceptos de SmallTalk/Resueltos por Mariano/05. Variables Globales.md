# Variables Globales en Pharo

En Pharo existen varios objetos globales ya creados por el sistema. Estos objetos sirven para interactuar con el entorno gráfico, mostrar mensajes, trabajar con archivos, etc.  
Las variables globales propias deben definirse previamente para no dar `nil`. Se definen así:

```smalltalk
NombreGlobal:= valor
```

---

## a.
```smalltalk
Display
```
**Devuelve:** `DisplayScreen(1920x1080x32)`  

**Explicación:**  
`Display` es un objeto global que representa la pantalla actual donde Pharo dibuja.  
Lo que ves entre paréntesis indica:
- **Ancho x Alto** de la pantalla en píxeles (por ejemplo 1920x1080).
- **Profundidad de color** en bits (por ejemplo 32).

> [!NOTE]
> Es obvio, pero, el valor que devuelva Pharo va a depender dependiendo de la pantalla actual.

---

## b.
```smalltalk
Transcript
```
**Devuelve:** `Transcript`  

**Explicación:**  
`Transcript` es un objeto global predefinido que representa la ventana de salida de texto de Pharo. Permite imprimir mensajes mientras el programa corre.

```smalltalk
Transcript show: 'Hola mundo'; flush.
```

---

## c.
```smalltalk
Debug
```
**Devuelve:** error (no definida en Pharo 8.0).  
**Explicación:**  

En Pharo 8.0 no existe un objeto global llamado `Debug` listo para usar.

---

## d.
```smalltalk
Disk
```
**Devuelve:** error (no definida en Pharo 8.0).  
**Explicación:**

Tampoco existe en Pharo 8.0 un objeto global `Disk` por defecto. Para manejar archivos/discos se usan clases como `FileSystem`, `FileReference` o `Directory`.

---

## e. y f.
```smalltalk
| Distancia |
Distancia := 15
```
**Explicación:**  
En Smalltalk, las barras verticales `| ... |` se usan para declarar variables temporales dentro de un bloque o método.  
Se pueden declarar varias a la vez:

```smalltalk
| x y z |
x:= 10.
y:= 20.
z:= x + y.
```

Son variables **locales** al contexto donde están declaradas, no son globales.

# COMPARACIÓN

## a.
```Smalltalk
3 < 4
```

**Devuelve:** `true`

El operador `<` compara para ver si la parte izquierda es menor a la derecha o lo que es lo mismo, que la parte derecha sea mayor a la izquierda)

---

## b.
```Smalltalk
#( 1 2 3 4) = #(1 2 3 4)
```

**Devuelve:** `true`

Ya vimos antes que el `=` compara igualdad entre dos partes.

---

## c.
```Smalltalk
'hola' <= 'adios'
```

**Devuelve:** `false`

En Pharo, los strings se comparan lexicográficamente, es decir, carácter por carácter según el valor ASCII de cada letra. Por ende, tan pronto como se encuentra un par de caracteres diferentes, la comparación se decide por esos caracteres y no se siguen comparando los siguientes.

Entonces, para este ejemplo:
1. Se compara el primer carácter de cada string: `'h' `(ASCII 104) y `a` (ASCII 97).
2. Como `104 > 97`, `'hola'` no es menor ni igual que `'adios'`.
3. Por lo tanto, la comparación devuelve `false`.

---

## d. y e.
```Smalltalk
5 = 2 + 3
```

Esto asi como esta va a devolver error porque se lee como `5 = 3` mensaje a 5 con argumento 3, y luego intenta aplicar + 2 sobre el resultado `(true o false) → error`, porque no podés hacer `true + 2.`. Esta correcto así:

```Smalltalk
5 = (2 + 3)
```

**Devuelve:** `true`

> [!NOTE]
> Los paréntesis son esenciales cuando se quiere que una operación se evalúe antes de enviar un mensaje.
> Esto pasa mucho con `=` y operaciones aritméticas, porque todo en Pharo son mensajes a objetos.

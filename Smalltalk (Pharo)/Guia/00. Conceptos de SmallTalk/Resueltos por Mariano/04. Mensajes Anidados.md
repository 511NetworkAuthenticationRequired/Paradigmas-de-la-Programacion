# MENSAJES ANIDADDOS

## a.  
```smalltalk
'hola' size + 4
```

**Devuelve:** `8`  
Suma el tamaño del string `'hola'` (4) con 4.  

---

## b.  
```smalltalk
'ahora' size + #(1 2 3 4) size
```

**Devuelve:** `9`  
Suma el tamaño de dos objetos distintos: la string `'ahora'` (5) y el array de 4 elementos → `5 + 4 = 9`.  

---

## c.  
```smalltalk
#(1 12 24 36) includes: 4 factorial
```

**Devuelve:** `true`  
Pregunta si el array contiene el factorial de 4 (4! = 24). Como 24 está en la posición 3, devuelve `true`.  

---

## d.  
```smalltalk
3 + 4 * 2
```  
**Devuelve:** `14` 
 
Pharo hace esto: primero suma `3 + 4 = 7`, luego Multiplica por `2 →  7 * 2 = 14`. 

> [!CAUTION]
> En Pharo todos los operadores binarios tienen la misma precedencia y se evalúan estrictamente de izquierda a derecha.
Esto se debe a que Smalltalk no implementa reglas matemáticas de jerarquía de operaciones, sino que trata cada operador como un mensaje binario más. Para obtener otro orden (como multiplicar antes de sumar) hay que usar paréntesis explícitamente.

---

## e.  
```smalltalk
3 + (4 * 2)
```

**Devuelve:** `11`  

Con paréntesis explícitos, la operación sigue la jerarquía: `4 * 2 = 8`, luego ` 3 + 8 = 11`.  

---

## f.  
```smalltalk
4 factorial between: 3 + 4 and: 'hola' size * 7
```

**Devuelve:** `true`  

Comprueba si el factorial de 4 (24) está entre `3 + 4 = 7 ` y `'hola' size * 7 = 28`. Como 24 está dentro del rango, devuelve `true`.  

---

## g.  
```smalltalk
'hola' at: (#(5 3 1) at: 2)
```

**Devuelve:** `'l'`  

Primero se evalúa el `at:` del array `( #(5 3 1) at: 2 ) = 3`. Luego `'hola' at: 3` devuelve `'l'`.  

---

## h.  
```smalltalk
(6 + 9) asString
```

**Devuelve:** `'15'`  

Convierte la suma de `6 + 9` en string. Sin los paréntesis, Pharo daría **error**.  

---

## i.  
```smalltalk
Array with: 1 with: 'hola' with: (1/3)
```

**Devuelve:** `#(1 'hola' (1/3))`  

Crea un Array de tamaño fijo con los tres elementos indicados. Es lo mismo que hacer `#(1 'hola (1/3))`
